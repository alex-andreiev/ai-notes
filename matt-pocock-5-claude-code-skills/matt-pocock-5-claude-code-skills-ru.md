# Конспект видео: 5 навыков Claude Code на каждый день (Matt Pocock)

## 1. Главный тезис
Для эффективного управления ИИ-агентом, например Claude Code, разработчику необходим четкий процесс — **Process-Driven Development**. Вместо хаотичных запросов Мэтт использует 5 строго структурированных кастомных команд («скиллов»), которые служат жесткими рельсами на разных этапах: от выравнивания идеи до рефакторинга архитектуры.

---

## 2. Пять базовых скиллов для ежедневной работы

### 1. `/grill-me` (Допрос)
* **Цель:** Исключить недопонимание (*misalignment*) на старте задачи.
* **Как работает:** Модель не начинает писать код, а агрессивно интервьюирует разработчика, задавая от 40 до 100 последовательных вопросов, проходя по всем ветвям проектных решений и выявляя скрытые зависимости.
* **Результат:** Формирование единого ментального понимания (*design concept*) перед началом работы.

### 2. `/write-a-prd` (Генерация требований)
* **Цель:** Зафиксировать результаты допроса в конечном документе назначении.
* **Как работает:** ИИ собирает все ответы из сессии `/grill-me` и упаковывает их в структурированный документ требований к продукту (PRD).
* **Результат:** Готовый markdown-файл с User Stories, техническими ограничениями, архитектурными решениями и критериями приемки.

### 3. `/prd-to-issues` (Разбиение на Kanban)
* **Цель:** Разбить монолитный PRD на независимые атомарные задачи.
* **Как работает:** Делит большую фичу на мелкие тикеты, строго следуя правилу **вертикальных срезов (Vertical Slices)**. Задачи укладываются в локальную Kanban-доску в виде markdown-файлов.
* **Результат:** Оптимальный пул задач, готовый для параллельной или последовательной работы ИИ-агентов в автономном режиме (AFK).

### 4. `/tdd` (Разработка через тестирование)
* **Цель:** Обеспечить контроль над кодом в процессе написания ИИ.
* **Как работает:** Заставляет ИИ строго следовать циклу Red-Green-Refactor. Модель сначала пишет падающий тест, запускает его, убеждается в ошибке и только потом внедряет реализацию.
* **Результат:** Надежная защита от «читерства» LLM и автоматическое покрытие системы качественными тестами.

### 5. `/improve-codebase-architecture` (Углубление модулей)
* **Цель:** Бороться с энтропией и хаотичным дроблением файлов.
* **Как работает:** Сканирует проект на наличие сильно связанных мелких компонентов (*shallow modules*) и предлагает пересобрать их в глубокие модули (*deep modules*) с мощной внутренней логикой и очень простым внешним интерфейсом.
* **Результат:** Кодовая база становится легко тестируемой, а интерфейсы — понятными для человека и ИИ.

---

## Главный вывод
Эти 5 скиллов превращают ИИ из непредсказуемого генератора текста в дисциплинированного, предсказуемого инженера. Задача человека — быть стратегическим архитектором: направлять `/grill-me`, контролировать интерфейсы в `/improve-codebase-architecture` и запускать петли обратной связи.
