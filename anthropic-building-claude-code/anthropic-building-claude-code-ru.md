# Конспект лекции: Как устроен Claude Code (Alex Albert, Anthropic)

## 1. Главный тезис лекции
Создание надежного CLI-агента для программирования (такого как `Claude Code`) требует кардинального переосмысления того, как модель взаимодействует с файловой системой и петлями обратной связи. Главный вызов — не просто научить модель генерировать текст, а минимизировать накладные расходы контекста, предотвратить зацикливание агента (*agent loops*) и обеспечить предсказуемое управление инструментами внутри терминала.

---

## 2. Архитектура работы с инструментами (Tool Use & Fast Feedback)
Алекс выделяет три ключевых инженерных столпа, на которых строится отзывчивость и точность Claude Code:

* **Специализированный тулинг (Bash vs Custom Tools):**
  * Вместо того чтобы давать модели неограниченный доступ к «сырому» Bash на каждый чих, Claude Code полагается на оптимизированные кастомные инструменты для чтения, поиска (grep) и редактирования файлов.
  * Родной Bash используется только тогда, когда модели абсолютно необходимо запустить внешние процессы (тесты, линтеры, сборку).
* **Мгновенные петли обратной связи (Instant Loops):**
  * Скорость работы агента зависит от того, насколько быстро он получает результат выполнения команды. Инженеры Anthropic оптимизировали потоковый вывод (streaming) и парсинг ошибок терминала, чтобы Claude мог мгновенно реагировать на упавший тест или ошибку компиляции, не дожидаясь генерации длинного ответа.

---

## 3. Проблема «Агентских циклов» и управление контекстом
Один из самых сложных failure modes для любого ИИ-агента — это бесконечные циклы (*infinite loops*), когда модель застревает на одной ошибке и раз за разом повторяет неверное действие.

* **Память и «забывание» лишнего:** Чтобы предотвратить раздувание контекста и переход модели в *Dumb Zone*, Claude Code использует внутренние механизмы очистки истории команд. Модель сохраняет только высокоуровневое дерево решений и текущую проблему, «выбрасывая» гигабайты логов предыдущих неудачных запусков тестов.
* **Раннее прерывание (Early Stopping):** Если агент совершает несколько однотипных правок подряд, система перехватывает управление и запрашивает подсказку у человека (*Human-in-the-loop*), возвращая разработчика к роли стратегического координатора.

---

## 4. Как оптимизировать свой проект под Claude Code
Чтобы терминальный агент работал в вашем репозитории максимально эффективно, Алекс Альберт даёт несколько практических рекомендаций:

1. **Используйте файл конфигурации (`claude.md`):** Опишите в корне проекта архитектурные правила, гайдлайны по кодстайлу и команды для запуска тестов. Это исключит фазу долгого «ощупывания» репозитория агентом.
2. **Пишите детерминированные тесты:** Если ваши тесты мигают (*flaky tests*), ИИ-агент сойдёт с ума, пытаясь починить то, что ломается случайным образом. Стабильные тесты — это единственный объективный критерий «правоты» для модели.
3. **Держите модули изолированными:** Чем меньше перекрёстных связей между файлами, тем меньше токенов Claude тратит на исследование зависимостей (паттерн глубоких модулей в действии).

---

## Главный вывод
`Claude Code` проектировался не как замена инженеру, а как его ультимативное продолжение в командной строке. Успех автоматизации кодинга зависит от синергии: Anthropic оптимизирует скорость работы и экономию контекста на уровне ядра модели, а разработчик организует чистую кодовую базу и жёсткие тестовые сценарии на уровне проекта.
