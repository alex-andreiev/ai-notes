# Конспект лекции: LIVE-сессия: Сборка проекта с нуля с помощью ИИ (Matt Pocock)

## 1. Главный тезис
Сухая теория воркфлоу ничего не стоит без жесткого стресс-теста в реальном времени. В этой сессии Мэтт демонстрирует, как запустить и разработать полноценное приложение с нуля, используя исключительно терминального ИИ-агента, сохраняя при этом идеальный контроль над качеством кода, типизацией и архитектурными границами проекта.

---

## 2. Анатомия запуска проекта «с нуля» (Greenfield Project)
Мэтт выделяет три важнейших шага на старте любого нового репозитория:

### 1. Первичная инициализация структуры (Scaffolding)
* Проект начинается не с кода, а с настройки жесткого фундамента. Первым делом разворачивается базовый стек, настраивается сборщик, конфигурируется TypeScript и инициализируется фреймворк тестирования (например, Vitest или Jest).
* **Правило:** Пока в пустом проекте не настроена автоматическая проверка типов (`tsc --noEmit`) и не проходит пустой тест, ИИ-агенту кодить запрещено.

### 2. Подготовка контекста проекта (`claude.md` / `.clauderc`)
* В корень проекта сразу помещается конфигурационный файл для ИИ-агента.
* В нём фиксируются технологический стек, правила форматирования кода, структура директорий и, самое главное, точные команды для запуска тестов и линтеров. Это исключает фазу "ощупывания" и холостую трату токенов моделью.

---

## 3. Практический цикл итераций в прямом эфире
В ходе стрима Мэтт показывает, как концепции из его коротких докладов работают «в боевом режиме»:

* Сначала scaffold, затем настройка и валидация, прежде чем писать код фичи.
* Держите ИИ-агента на жёстком поводке, заставляя его выполнять проверки типов и тесты с самого начала.
* Используйте конфигурационный файл, чтобы дать агенту детерминированный контракт проекта.

---

## Главный вывод
LIVE-сессия доказывает, что разработка с помощью ИИ работает только тогда, когда проект с самого первого дня создан для агента. Чистая настройка, строгая валидация и общий контракт проекта — основа надёжных workflows для терминальных агентов.
